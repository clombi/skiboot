      21                 :            :  *
      22                 :            :  * LPC->AHB bridge
      23                 :            :  * ---------------
      24                 :            :  *
      25                 :            :  * This bridge directly converts memory or firmware accesses using
      26                 :            :  * a set of registers for establishing a remapping window. We prefer
      27                 :            :  * using FW space as normal memory space is limited to byte accesses

      29                 :            :  * access sizes and to control the IDSEL bits which essentially enable
      30                 :            :  * a full 4G addres space.
      31                 :            :  *
      32                 :            :  * The way FW accesses map onto AHB is controlled via two registers
      33                 :            :  * in the BMC's LPC host controller:
      34                 :            :  *
      35                 :            :  * HICR7 at 0x1e789088 [31:16] : ADRBASE
      36                 :            :  *                     [15:00] : HWMBASE
      37                 :            :  *
      38                 :            :  * HICR8 at 0x1e78908c [31:16] : ADRMASK
      39                 :            :  *                     [15:00] : HWNCARE
      40                 :            :  *
      41                 :            :  * All decoding/remapping happens on the top 16 bits of the LPC address
      42                 :            :  * named LPC_ADDR as follow:
      43                 :            :  *
      44                 :            :  *  - For decoding, LPC_ADDR bits are compared with HWMBASE if the
      45                 :            :  *    corresponding bit in HWNCARE is 0.
      46                 :            :  *
      47                 :            :  *  - For remapping, the AHB address is constructed by taking bits
      48                 :            :  *    from LPC_ADDR if the corresponding bit in ADRMASK is 0 or in
      49                 :            :  *    ADRBASE if the corresponding bit in ADRMASK is 1
      50                 :            :  *
      51                 :            :  * Example of 2MB SPI flash, LPC 0xFCE00000~0xFCFFFFFF onto
      52                 :            :  *                           AHB 0x30000000~0x301FFFFF (SPI flash)
      53                 :            :  *
      54                 :            :  * ADRBASE=0x3000 HWMBASE=0xFCE0
      55                 :            :  * ADRMASK=0xFFE0 HWNCARE=0x001F
      56                 :            :  *
      57                 :            :  * This comes pre-configured by the BMC or HostBoot to access the PNOR
      58                 :            :  * flash from IDSEL 0 as follow:
      59                 :            :  *
      60                 :            :  * ADRBASE=0x3000 HWMBASE=0x0e00 for 32MB
      61                 :            :  * ADRMASK=0xfe00 HWNCARE=0x01ff
      62                 :            :  *
      63                 :            :  * Which means mapping of   LPC 0x0e000000..0x0fffffff onto
      64                 :            :  *                          AHB 0x30000000..0x31ffffff

      66                 :            :  * iLPC->AHB bridge

      68                 :            :  *
      69                 :            :  * This bridge is hosted in the SuperIO part of the BMC and is
